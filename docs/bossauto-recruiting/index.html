install

一条命令安装 Agent Skills

安装器只会安装仓库 skills/ 下包含 SKILL.md 的 Skill 目录，不会安装本地测试结果、浏览器登录状态、运行时证据、录屏素材或仓库工具。

公开 npm 安装器

推荐方式

npm install -g @zyanwan/patchright-agent-installer --registry=https://registry.npmjs.org/
patchright-agent-installer install codex

适合同事或用户从公开 npm 获取安装器，再从私有 Skill 源安装到本机 Agent 目录。

私有 GitHub 源

直接安装

npx -y git+https://github.com/ZyanWan/Patchright-Agent.git install codex

适合已经拥有私有仓库读取权限的用户，直接从仓库下载并安装内置的 skills/。

Codex: ~/.agents/skills Claude: ~/.claude/skills 自定义: --target <path>

workflow

先观察，再判断，最后才行动

这个项目不是一组孤立点击脚本，而是一条围绕证据和审计设计的招聘流水线。默认路径保持只读，任何网站状态变更都需要当前授权和目标复核。

  1. 01

    页面诊断

    识别当前 BOSS 页面类型、URL、frame 和候选人卡片结构，保留用户当前业务上下文。

  2. 02

    候选人扫描

    提取 stableId、姓名、公司、岗位、学历、全文卡片证据和疑似已沟通信号。

  3. 03

    规则快筛

    按招聘要求做本地 dry-run，信息不足或身份不稳时进入 review，而不是强行拒绝。

  4. 04

    详情证据

    打开候选人详情，保存 DOM 文本、截图、canvas 诊断和身份匹配状态。

  5. 05

    受控动作

    收藏、发起沟通和批量沟通默认 dry-run，只有明确授权后才按 stableId 重新定位并执行。

  6. 06

    审计交付

    输出 JSON、截图、decision records 和 append-only action log，让招聘负责人可以复核。

guardrails

把高风险动作留在边界内

默认允许

  • 打开可见浏览器，交给用户完成登录
  • 读取标题、URL、frame、候选人卡片
  • 保存截图、JSON 和详情证据
  • 本地记录筛选结果和人工复核原因

必须授权

  • 收藏、保存、发起沟通或批量动作
  • 应用筛选条件或改变页面状态
  • 切换职位、刷新推荐列表或处理已沟通候选人
  • 任何可能触发付费、限制或不可逆状态的操作
不做的事 不绕过登录、验证码、MFA、限制或付费控制。 不自动编写和发送自定义聊天消息。 不靠列表序号点击候选人。

runtime artifacts

每一步都有本地证据

页面结果不是口头承诺，而是可解析、可追踪的工作目录。后续做报告、Excel 或人工复核时，可以直接复用这些文件。

candidates.json

候选人扫描

当前页面可见卡片、stableId、frame 来源、疑似已沟通标记和完整卡片文本。

detail-evidence/*.png

详情截图

对 DOM 文本不足、canvas 简历或身份不确定的候选人，保留可视化证据。

dry-run-decisions.json

快筛判断

区分 pass、reject、review、skip，不把信息缺失误当成硬拒绝。

action-log.jsonl

动作审计

记录每次授权动作的候选人、原因、状态、时间戳和证据路径。

# 筛选判定 v2 迭代 — 交接文档

> 目的:把"详情判定"从"看裸数字直接打分"升级为「先抽离(主体/时间/数据口径)→ 归一化 → 按主体门槛判断」;并修复公司关误刷、详情读取串人/编造。
> 判定始终保持:**二选一(收藏=通过 / 不要=刷)、拿不准就过、模拟优先**。
> 本文自包含,实施者无需原始对话上下文。

---

## 0. 一句话
详情判定要先做结构化抽离再归一化对标;公司关把确定性名单从 LLM 手里收回代码;详情正文要留存可核、防串人。

## 1. 项目背景
- 应用:`bossauto-electron`(Electron + patchright),BOSS 直聘招聘端自动筛简历。
- 漏斗顺序:扫列表卡 → 去重/跳过已沟通 → **硬筛**(确定性:学历/年龄/经验/城市/985 等)→ **公司关**(LLM,只看公司)→ **职位关**(LLM,只看职位/经历标题)→ 开详情 → **详情判定**(LLM:优势 + 工作经历 + 项目经验)→ 收藏/不要 →(非模拟时)打招呼。
- LLM:DeepSeek(OpenAI 兼容协议)。详情判定、公司关、职位关都走它。
- 目标人群:很强的短视频内容操盘手(**亲手做**:脚本/拍摄/剪辑/发布),在竞争性内容赛道任过职(MCN/自媒体、线上教育、内容/直播电商),业绩好。
- 对口门槛(已在 criteria,保持):必须亲手做短视频;排除纯图文、达人运营/BD、纯剪辑、纯投放、产品/活动运营、纯团队管理;必须有"公司任职做短视频"经历(不能纯个人号 freelance)。

## 2. 当前状态(已完成,勿重复)
- 详情判定已是**二选一**(收藏/不要),无"待定";**拿不准就过**已在详情 prompt 生效。
- 多项目结构:配置在 `%AppData%/bossauto-electron/projects/<id>/criteria.yaml`;**当前活动项目=「自媒体」**(id 见 `projects/index.json`),业绩标准已写入该项目。
- 模拟模式 `dry_run: true` 默认开:判定但不在 BOSS 真点(不打招呼/不收藏),只记录。验证阶段保持开启。
- 自动启动:`BSA_AUTO=1 BSA_AUTO_RUN=1 npm run dev` → 自动附加已开的 Chrome(端口 9222)、自动开跑。**复用现有登录,别新开浏览器**(同账号双登录会顶号)。

## 3. 发现的问题(要修)
### 3.1 公司关 LLM 误刷(把想要的也刷了)
- 字节(在允许名单)被刷;淡蓝网 Blued(小规模垂直,应放过)被刷;中国音网"疑似"就刷(违反拿不准就过);亿邦动力被错认成"官媒"。
- 根因:LLM 把"除字节/快手/小红书外的大厂/中厂都刷"**读反**成"只留这三家、其余互联网都刷";不认"小规模垂直不因规模过滤";疑似就刷;事实分类错。**是 LLM 判断问题,不是代码 bug**。

### 3.2 详情读取疑似串人/编造
- 例:候选人「唐思维」判定理由出现"千万播放爆款 + 月均 300 线索、成交率 70%",像是两个人(内容 + 销售)的数据被拼一起,或 LLM 编造。
- 详情正文是用「框选一屏 + Ctrl+C 读剪贴板」拿的(BOSS 简历是 **wasm 加密 canvas 渲染**;网页直读 DOM、canvas `fillText` hook、`getSelection` **都拿不到** → 这两条是死路)。
- 现在**没存**喂给 LLM 的正文(只记了字数),无法核到底串没串。

### 3.3 判定太宽松(没归一化)
- 看裸数字直接打分,没结合时间/数据口径/主体类型。导致"2 年累计才几十万播放""甲方一年多涨粉才一万多"也被判过。

## 4. 要做的改动

### A. 详情读取清理(`src/main/lib/runner.ts`)
- **删掉两条死路**:
  - 网页直读:`readDomDetailText`
  - canvas hook:`CANVAS_HOOK_SCRIPT`、`ensureCanvasHook`、`clearCaptured`、`readCanvasFromAllFrames`,以及它们在 `processFunnel` 详情循环里的调用。
  - `readDetailText` 里对 DOM/hook 文本的合并。
- **只保留框选复制** `readCanvasClipboardText`;`readDetailText` 直接返回它即可。
- `mergeTextChunks`(框选复制用于拼接滚动分段)**保留**。
- `dumpDetailDom`(canvas 诊断)可一并删。
- typecheck 注意:删函数后清理无用 import。

### B. 正文留存 + 姓名核对(`runner.ts` 的 `processFunnel` 详情循环)
- 每个详情读完,把**实际喂给 LLM 的正文**存盘:如 `%AppData%/bossauto-electron/detail-dumps/<序号>-<姓名>.txt`(含时间戳)。
- **姓名核对**:正文里若不含候选人姓名 → 记警告(疑似框错/详情没切换干净),可重读一次;仍不含则在日志标"存疑"(按拿不准就过仍可放过,但要可见)。
- 目的:可核"搞错人",并给"抽离"留底。

### C. 新判定逻辑(重写 `src/main/lib/llm.ts` 详情 `softJudge` 的 SYSTEM prompt)
一次 LLM 调用,输出结构化结果。分两段:

**(1) 抽离 —— 只提取事实,先不下结论**
1. **主体类型**(每段相关经历):`自己搞` / `MCN·自媒体机构` / `甲方(品牌自有号)` / `乙方(代运营·服务客户)`
2. **时间**:每段相关经历的起止 + 时长(简历有标准日期格式,正文里有,务必抽出)
3. **数据口径**:每个数字标注 —— 指标(涨粉/播放/赞藏/GMV/线索)×范围(**单条视频 / 单月 / 单账号累计 / 全年 / 整个生涯累计**)×时间窗

**(2) 归一化判断 —— 结合上面三项对标**
- 把裸数字换算成"**代表作水平 + 单位时间产出**"再对标,**不看绝对数**。
- **门槛排序与倍数**:
  - `自己搞` = 基线:一年内约 **1 万粉**,或出过 **几千赞藏** 的爆款(且必须可复制/非孤例)。
  - 前三类 = 基线 **×5~10 倍**,高到低:**MCN(~10x,最高) > 甲方(~7x,中) > 乙方(~5x,最低)**。
    - 乙方:约 5 万粉/年 或单条数万赞藏;甲方:约 7–10 万粉/年;MCN:约 10 万+/年 或多条数万赞藏 / 千万播放级爆款。
  - 倍数是区间锚点,可按 criteria 微调。
- **赞藏为主**;播放不设绝对硬线,但用于归一化(明显过低如"2 年累计几十万播放"要刷)。
- 接手已有基础的号:涨粉不算(基数撑的),看任内有无**持续**做出过万赞藏爆款。
- 做短视频满一年仍无任何达标成绩 → 刷。
- **固化"硬不行"例子**:
  - 甲方 · 一年多 · 涨粉才一万多 → 刷
  - 任意主体 · 1–2 年 · 累计才几十万播放 → 刷
  - 全靠单条/单月数据撑场、无持续累计量级 → 偏弱
- **仍二选一 + 拿不准就过**:对口、有亮点但时间/数据缺失或是否达标模糊 → 判收藏放过,交人工。

**(3) 输出(JSON)**:`{ 抽离结果(主体/时间/数据口径), score(1-10), d: "收藏"|"不要", r }`。
- `r` 必须点出 **主体类型 + 时间跨度 + 数据口径 + 归一化结论**。例:"甲方 2 年,账号累计 30 万播放,摊到月偏低,刷"。

### D. 公司关名单快筛 + LLM 兜底(`src/main/lib/judge.ts` / `runner.ts` + criteria)
- criteria 增加**确定性名单**(数组,放业务配置不硬编码代码):
  - 禁止大厂(子串匹配):阿里、腾讯、美团、京东、拼多多、百度、B站、微博、小米、网易、滴滴、知乎、携程、苏宁、叮咚、Coupang …
  - 禁止体制内关键词:政府、事业单位、出版社、学报、大学、研究院、日报、周刊、电视台、广播、报社、杂志社 …
  - 允许名单:字节、快手、小红书
- 代码(硬筛/公司关之前对每张卡的公司文本做匹配):
  - 命中**禁止** → 直接刷,**不进 LLM**
  - 命中**允许** → 直接放过,**不进 LLM**
  - 都没命中 → 才送公司关 LLM
- 公司关/职位关的 LLM prompt(`gateSystemPrompt`)**补上"拿不准就过/疑似不刷"**(目前只有详情判定有这条)。

## 5. 全局原则与约束(必须遵守)
- 判定二选一(收藏=通过 / 不要=刷),**无待定**。
- **拿不准就过**:只在"明确不对口 / 全文零业绩 / 有数据但明确低于线 / 满一年没成绩"才刷;其余放过交人工。宁可多放过,不可误刷。
- 通过的动作是"打招呼"(任务 `action: greet`);但 `dry_run` 下只记录不真点。**验证阶段保持 `dry_run: true`**。
- 浏览器复用已开的 Chrome(9222),别新开;同账号双登录会顶号。
- 业务规则放 criteria(配置),代码只执行,不在代码里硬编码业务标准(名单也放 criteria)。
- 全程中文回复用户;不向用户展示代码/命令/日志原文。
